# kamishibai

TRPGリプレイ動画（紙芝居動画）クリエイター。
紙芝居クリエーターで手作業していた立ち絵・メッセージウィンドウ・ステータス表示の組み立てを、
テキスト脚本から自動生成するWebアプリです。まずは迷宮キングダム向け。

## 起動

```sh
npm install
npm run dev
```

## 使い方

1. **素材ライブラリ**に立ち絵・背景・BGM・SE・ダイス連番画像を登録
   （**フォルダごとドラッグ&ドロップ／フォルダ選択**に対応。フォルダ構造はそのまま保持されます）
2. **キャラクター**を追加し、表情差分の画像・パラメータ初期値（HP/気力/配下/状態）・
   立ち絵の**表示倍率／縦位置**（素材の解像度差の補正）・**顔アイコンとその倍率**・
   **マップチップ画像とその倍率**・**ダイスセット**（連番画像のフォルダ）を設定
3. **脚本**を書くと右側にプレビューが組み上がる

ダイスは `@dice` で連番アニメが転がってから出目を表示します。ダイス素材には
**フォルダ**（中の画像をファイル名順に連番アニメとして再生）か**画像1枚**
（転がしなしの静止ダイス）を指定できます。ヘッダーの「ダイスアニメ」チェックを
外すと、常にアニメなしで即座に出目だけ表示するモードになります。
キャラにダイスが未設定の場合は、ヘッダーで選んだ既定ダイスが使われます。

素材の選択はすべて「フォルダ → ファイル」の2段選択です。さらにキャラクターに
**素材フォルダを紐づける**と、そのキャラの立ち絵・アイコン等の選択候補が
紐づけたフォルダ内だけに絞られます（未紐づけなら全素材から選択）。

**素材はリポジトリに含めません（コミット禁止）。** 素材はローカルPCから毎回登録する
運用です（IndexedDBによるブラウザ内永続化をM2で対応予定）。

脚本とキャラ設定はブラウザに自動保存されます。バックアップや別マシンへの移行は
ヘッダーの「保存（書き出し）」「開く（読み込み）」でプロジェクトファイル（.json）として
やり取りできます（素材ファイルは含まれないため、別途登録し直してください）。

### 脚本の記法

```
@bg 洞窟.png              # 背景変更
@bgm 戦闘.mp3             # BGM（@bgm stop で停止）
@se ダイス.wav            # 効果音
@show 名前 表情 left      # 立ち絵表示（セリフを書けば空いた位置に自動表示されます）
@show 名前 right flip     # 右側に左右反転で表示（flip/noflip。向かい合わせの演出）
@hide 名前                # 立ち絵消去
@damage 名前 5            # ダメージ → ステータスバーのHPに自動反映＋ポップ演出
@heal 名前 3              # 回復
@set 名前 状態 肥満       # キャラの任意パラメータ変更
@setglobal ターン 3       # 全体パラメータ（民の声・ターン等）の変更
@dice 2d6 8               # ダイス演出（連番アニメ→出目表示）
@dice 名前 2d6 8          # キャラのダイスセット（色）で振る
@map 戦場.png             # 画像の戦闘マップ/ダンジョンマップ表示（@map off で消去）
@bf 地雷原 地雷原 敵前衛 敵後衛   # 素材不要の生成戦場マップ（列ラベルを並べる。@bf off で消去）
@lane 2 danger            # 戦場トラップ発動：列2が赤く変わる（normal で解除）
@lane 2 罠原              # 列2のラベルを変更（状態と同時指定も可: @lane 2 罠原 danger）
@chip 名前 30 60          # 画像マップ: %座標(0-100) / 生成マップ: 列 行（小数で微調整可）
@chip 名前 off            # チップ撤去
@mark 3 2 死              # 白札マーカー（死・天・鴉など。@mark 3 2 off で撤去）
@status off               # ステータスバー表示切替
@wait 1.5                 # 表示時間（秒）

名前: セリフ              # 1カット。立ち絵は自動表示（複数人は左→右→中央に自動配置）
名前(笑顔): セリフ        # 表情差分を自動差し替え。発言していないキャラは自動で暗くなります
```

設計の詳細は [docs/DESIGN.md](docs/DESIGN.md) を参照。
