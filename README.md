# kamishibai

TRPGリプレイ動画（紙芝居動画）クリエイター。
紙芝居クリエーターで手作業していた立ち絵・メッセージウィンドウ・ステータス表示の組み立てを、
**テキスト脚本から自動生成**するWebアプリです。まずは迷宮キングダム向け。

> **📖 はじめての方・使い方を知りたい方は → [使い方ガイド（docs/GUIDE.md）](docs/GUIDE.md)**
> 全体像・チュートリアル・やりたいこと別のレシピ・全コマンドを、順番に読めるようにまとめています。

## 起動

```sh
npm install
npm run dev
```

## 5つのステップ

```
① 素材を登録  →  ② キャラを作る  →  ③ 脚本を書く  →  ④ プレビュー  →  ⑤ mp4に書き出す
```

1. **素材ライブラリ**に立ち絵・背景・BGM・SE・ダイス連番画像を登録
   （フォルダごとドラッグ&ドロップ／フォルダ選択に対応。フォルダ構造は保持されます）
2. **キャラクター**を追加し、表情差分の立ち絵・パラメータ初期値・各種倍率・ダイスセットを設定
3. **脚本**を書くと右側にプレビューが組み上がる
4. `▶ 再生` で通しの動きを確認
5. ヘッダーの **🎬 動画書き出し** でmp4（1280×720 / 30fps、BGM/SE込み）としてダウンロード
   （WebCodecs対応ブラウザ＝Chrome/Edge推奨）

## 最短の例

```
@bg 洞窟.png
@bgm 戦闘.mp3

国王: ここが例の迷宮ね。
従者(笑顔): わくわくしますねっ！
@damage 国王 5
国王(苦悶): くっ……いきなり罠！？
```

- `名前: セリフ` が1カット。立ち絵は自動配置され、`名前(表情):` で差し替わります。
- `@` で始まる行が演出命令（背景・音・ダメージ・戦場・フェードなど）。
- ダメージはステータスバーに自動反映されます。

書き方の全体は **[使い方ガイド](docs/GUIDE.md)** に、内部設計は [DESIGN.md](docs/DESIGN.md) にあります。

## 素材の扱い（重要）

**素材（画像・音声）はリポジトリに含めません（コミット禁止）。**
登録した素材はブラウザ内（IndexedDB）に保存され、次回起動時に自動で復元されます。
素材ライブラリの「すべて削除」で保存データごとリセットできます。
※ブラウザやマシンをまたぐ移行は、素材フォルダを再登録してください。

脚本とキャラ設定はブラウザに自動保存されます。バックアップや別マシンへの移行は
ヘッダーの「保存（書き出し）」「開く（読み込み）」でプロジェクトファイル（.json）として
やり取りできます（素材ファイルは含まれないため、別途登録し直してください）。

## フォント

エディタと動画のフォントは、既定でユニバーサルデザインの **BIZ UDPゴシック**
（モリサワ製、SIL OFLライセンスで同梱）を使います。動画のフォントは
ヘッダーの選択で変更できます（プロジェクトファイルに保存されます）。

## ロードマップ

- [x] M1: 脚本→プレビュー、キャラDB、ダメージ自動処理
- [x] M2a: フォルダ一括登録、ダイス連番アニメ（キャラ別ダイス・OFFモード）、立ち絵の倍率補正
- [x] M2b-1: 素材のIndexedDB永続化
- [x] M2b-2: BGM/SE本格対応（音声タイムライン・フェード・音量・ミキサー）
- [x] M3: mp4書き出し（WebCodecs + mp4-muxer、BGM/SE込み）
- [ ] M4: レイアウト編集、ゲームテンプレートエディタ（迷キン以外への対応）
