# kamishibai — TRPGリプレイ動画クリエイター 設計書

## 目的

紙芝居クリエーター（Windows）で手作業により行っていたTRPGリプレイ動画
（立ち絵＋メッセージウィンドウ＋ステータス表示のいわゆる「紙芝居動画」）の制作を、
モダンなUIで大幅に省力化するWebアプリ。

### 従来の課題

- レイヤー切り・画像挿入・キャラパラメータをすべて手打ちしていた
- キャラの表情差分の差し替えが「大量の素材から手で選ぶ」作業だった
- ダメージ処理のたびにステータス表示を手で書き換えていた

## 決定事項

| 項目 | 決定 | 理由 |
|------|------|------|
| プラットフォーム | Webアプリ（ブラウザ） | Mac/Win両対応・インストール不要・開発が速い |
| 動画出力 | mp4直接書き出し | WebCodecs + mp4-muxer。BGM/SE込みで完結 |
| 編集スタイル | テキスト脚本＋GUI補助 | ベタ打ちの執筆速度を維持しつつ自動組版 |
| ゲームシステム | まず迷宮キングダム固定 | ただし内部は GameTemplate で抽象化し、後で差し替え可能に |

## アーキテクチャ

```
脚本テキスト ──parse──▶ ScriptCommand[] ──fold──▶ Cut[]（各カットの描画状態）
                                                    │
                              ┌─────────────────────┤
                              ▼                     ▼
                        プレビュー再生          mp4書き出し
                        （Canvas描画）       （同じ draw() を
                                              フレームごとに実行）
```

- **描画は純関数** `drawCut(ctx, cut, assets, template)` に集約する。
  プレビューと動画書き出しが同じコードパスを通るため「プレビューと完成品がズレない」。
- **GameTemplate** がゲーム固有の定義をすべて持つ。エンジン（パーサ・カット展開・描画）は
  テンプレートを参照するだけで、迷キン固有の値をコードに持たない：
  - キャラパラメータのスキーマ（HP/気力/配下/状態…）と全体パネル（民の声/ターン…）
  - `@damage` が増減させるパラメータ（damageParamKey）
  - 生成戦場マップの標準列・行数・敵味方判定キーワード（battlefield）
  - プロジェクトには templateId を保存（現状 maze-kingdom のみ）

## 脚本フォーマット

```
@bg 森.png                     # 背景変更
@bgm 戦闘.mp3                  # BGM開始（stop で停止）
@se ダイス.wav                 # 効果音
@show 国王 笑顔 left     # 立ち絵表示（表情・位置は省略可）
@hide 国王               # 立ち絵消去
@damage 国王 5           # ダメージ処理：HP自動減算＋ダメージポップ表示
@heal 国王 3             # 回復
@set 国王 状態 肥満      # 任意パラメータ変更
@dice 2d6 8                    # ダイス演出（連続絵アニメ＋出目表示）
@status off                    # ステータスバー表示切替（on/off）
@wait 1.5                      # 待機（秒）

国王: そんなんですませるかぁ！！
国王(怒り): キョーレツなのを撃ってやるっ！！   # 表情差分を指定
GM: 洞窟の奥から地響きが聞こえる……                    # 未登録名も話者として使える
```

- `名前: セリフ` の行が1カット。コマンド行（`@`）は次のセリフ行に演出として乗る。
- `名前(表情):` でそのカットだけ立ち絵の表情差分を自動差し替え。
- `#` 以降はコメント。

## データモデル（要約）

- `GameTemplate` — パラメータ定義（pair=現在値/最大値、number、text）とレイアウト
- `Character` — 名前、表情差分画像（表情名→アセット）、パラメータ現在値
- `Project` — テンプレート、キャラ一覧、脚本テキスト、グローバルパラメータ
- `Cut` — 背景、表示中立絵、パラメータスナップショット、メッセージ、演出（ダメージ/ダイス/SE）

ダメージ・回復・@set はカット展開時にパラメータへ畳み込まれるので、
以降のカットのステータスバーには自動で反映される（手打ち書き換えの廃止）。

## 音声アーキテクチャ

```
Cut[] ──buildAudioTimeline──▶ AudioTimeline（BGM区間 + SEイベント + 総尺）
                                   │
                   ┌───────────────┤
                   ▼               ▼
            プレビュー再生    renderAudioTimeline
            （HTMLAudio）    （OfflineAudioContext → AudioBuffer、mp4書き出し用）
```

- 各カットの開始時刻は `cutTimings()`（wait指定 or 既定2.5秒の累積）で決定的に決まり、
  映像フレームと音声の同期の唯一の基準になる
- `AudioTimeline` はBGMの連続区間（開始/終了秒・音量・フェードイン/アウト）と
  SEイベント（発火秒・音量）を持つ。プレビューと書き出しが同じ表現を共有するため、
  「プレビューで聞いた通りの音」が動画になる
- 単体テスト（vitest）でタイムライン導出を固定済み

## 素材の扱い

- 素材（立ち絵・背景・BGM・SE）はユーザーのPCからファイル選択/ドラッグ&ドロップで登録
- ブラウザ内では IndexedDB に blob として永続化（予定）。MVPはメモリ保持
- 脚本からはファイル名で参照

## 画面構成（MVP）

```
┌───────────────┬──────────────────────┐
│ 脚本エディタ   │  プレビュー（Canvas 1280x720）│
│（テキスト）    │  ◀ ▶ 再生コントロール          │
├───────────────┼──────────────────────┤
│ キャラクター管理（追加・画像割当・パラメータ初期値）│
│ 素材ライブラリ                                     │
└──────────────────────────────────────┘
```

## ロードマップ

1. **M1（MVP）**: 脚本パース→カット再生プレビュー、キャラDB、ダメージ自動処理、立ち絵/背景/メッセージ描画
2. **M2**: BGM/SE再生、ダイス連続絵アニメ、IndexedDB永続化、プロジェクト保存/読込（JSONエクスポート）
3. ~~M3: mp4書き出し~~ 完了（WebCodecs + mp4-muxer、OfflineAudioContextで音声ミックス）
4. **M4**: レイアウト編集（ステータスバー・ウィンドウの位置/デザイン変更）、テンプレートエディタ（他ゲーム対応）
